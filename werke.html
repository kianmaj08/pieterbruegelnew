Wichtige Werke mit Kurzanalysen

Die niederländischen Sprichwörter (1559)

Ein Wimmelbild der Redensarten. Mehr als hundert Sprichwörter werden als kleine Szenen sichtbar gemacht. Das Bild wirkt lustig, ist aber eine ernste Kritik am menschlichen Verhalten.

Kinderspiele (1560)

Über zweihundert Kinder bei Spielen. Neben Spaß zeigen einige Spiele auch Konkurrenz, Täuschung und Gewalt. Deutung: Das Leben ist ein Spiel mit Regeln, die man lernen muss.

Kampf zwischen Karneval und Fasten (1559)

Gegenüberstellung von Ausgelassenheit und religiöser Strenge. Bruegel zeigt beide Seiten ohne eindeutige Partei. Das Bild ist eine soziale Beobachtung und eine Frage nach Maß und Mitte.

Bauerntanz (um 1567)

Dorffest mit Musik und Tanz. Rhythmus, Alkohol und Flirten werden sichtbar. Neben der Freude blitzt Kritik an Maßlosigkeit auf.

Bauernhochzeit (1567/68)

Eine Hochzeit, bei der das Essen, die Organisation und die Rollenverteilung genau zu erkennen sind. Der Bräutigam ist kaum im Zentrum, stattdessen das gemeinsame Fest. Das Bild ist eine Quelle für Alltagskultur.

Turmbau zu Babel (1563)

Ein gigantischer, aber brüchiger Bau. Symbol für Überheblichkeit und scheiternde Großprojekte. Bruegel zeigt technische Details und die Logik des Bauens – und gleichzeitig das sichere Scheitern.

Jahreszeiten (1565)

Mehrteilige Folge: Jäger im Schnee, Kornernte, Heuernte, Rückkehr der Herde und weitere. Die Bilder verbinden Wetter, Arbeit und Freizeit. Sie sind Meilensteine der europäischen Landschaftsmalerei.

Triumph des Todes (um 1562/63)

Eine Endzeitvision: Skelette besiegen die Menschen aller Stände. Das Bild erinnert an die Unsicherheit der Zeit und an die Vergänglichkeit allen Lebens.

Dulle Griet (um 1563)

Eine furchtlose Frau stürmt Richtung Hölle und plündert. Gier, Angst und Chaos werden grotesk dargestellt. Die genaue Botschaft bleibt offen, wirkt aber wie eine Warnung.

Kreuztragung Christi (1564)

Christus ist nicht isoliert, sondern Teil einer großen Volksmenge in niederländischer Landschaft. Das lenkt den Blick auf die Reaktionen der Menschen und auf die Frage nach Mitgefühl.

Predigt Johannes des Täufers (1566)

Eine Menschenmenge hört dem Prediger zu. Bruegel interessiert die Wirkung einer Botschaft auf verschiedene Gruppen. Kleidung, Haltung und Abstände erzählen vom sozialen Gefüge.

Hintergrund: Integration einer bestehenden React-Komponente (shadcn / Tailwind / TypeScript)

Du bist damit beauftragt, eine bestehende React-Komponente in eine Codebasis zu integrieren. Die Codebasis soll folgende Anforderungen unterstützen:

  • shadcn-Projektstruktur
  • Tailwind CSS
  • TypeScript

Falls die Basis das nicht unterstützt, befolge die Setup-Schritte unten (shadcn CLI, Tailwind, TypeScript). Bestimme außerdem den Standardpfad für Komponenten und Styles. Liegt der Default-Pfad nicht unter /components/ui, lege diesen Ordner an und begründe kurz, warum die Bündelung dort sinnvoll ist (klare Importe, Konsistenz, einfache Migration).

Ergänze deine tailwind.config.js um Folgendes:

const {
  default: flattenColorPalette,
} = require("tailwindcss/lib/util/flattenColorPalette");

/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    // your paths
    "./src/**/*.{ts,tsx}",
  ],
  darkMode: "class",
  theme: {
    extend: {
      animation: {
        aurora: "aurora 60s linear infinite",
      },
      keyframes: {
        aurora: {
          from: {
            backgroundPosition: "50% 50%, 50% 50%",
          },
          to: {
            backgroundPosition: "350% 50%, 350% 50%",
          },
        },
      },
    },
  },
  plugins: [addVariablesForColors],
};

// This plugin adds each Tailwind color as a global CSS variable, e.g. var(--gray-200).
function addVariablesForColors({ addBase, theme }: any) {
  let allColors = flattenColorPalette(theme("colors"));
  let newVars = Object.fromEntries(
    Object.entries(allColors).map(([key, val]) => [`--${key}`, val])
  );

  addBase({
    ":root": newVars,
  });
}

Implementation Guidelines

  1. Analysiere die Komponentenstruktur und alle Abhängigkeiten.
  2. Prüfe Props/Argumente und etwaigen internen State.
  3. Installiere nötige Provider/Hooks (z. B. ThemeProvider, sofern genutzt).
  4. Fragen:
    • Welche Daten/Props werden an die Komponente übergeben?
    • Gibt es State-Management-Anforderungen?
    • Werden Assets (Bilder, Icons) benötigt?
    • Wie soll die Komponente responsiv reagieren?
    • Wo im App-Layout ist der beste Einsatzort?

Schritte zur Integration

  1. Code wie oben in die richtigen Ordner kopieren (insb. /components/ui).
  2. Externe Dependencies installieren: 
    npm i tailwindcss postcss autoprefixer framer-motion lucide-react
npm i -D typescript @types/react @types/node
# Falls shadcn noch nicht konfiguriert ist:
npx shadcn@latest init
  3. Bild-Assets zunächst mit Unsplash-Stockfotos füllen (Platzhalter).
  4. lucide-react Icons verwenden, wenn simple SVG-Logos benötigt werden.

Wenn das Projekt shadcn/Tailwind/TS noch nicht hat

Kurzanleitung:

  1. TypeScript: npm i -D typescript @types/react @types/node, dann tsconfig.json erzeugen (Next.js: via next dev automatisch).
  2. Tailwind: npx tailwindcss init -p, Pfade in tailwind.config.js setzen, globals.css mit Tailwind-Direktiven (@tailwind base; @tailwind components; @tailwind utilities;).
  3. shadcn: npx shadcn@latest init; Standard-Komponentenpfad prüfen. Wenn nicht /components/ui, Ordner anlegen und wegen Konsistenz/Austauschbarkeit dort bündeln.

Standardpfade

  • Komponenten: /components/ui (empfohlen für shadcn-Struktur).
  • Styles: /app/globals.css (Next.js) bzw. src/styles/globals.css in CRA/Vite-Varianten.
